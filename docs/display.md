# display.html

**Owner:** Person B

**What it does:** This is the host screen — the page the person running the server opens on the laptop. It shows the QR code players scan to join, the live player list, the Start Game button, and the current game state as rounds progress.

**Your job:** Same as index.html — the logic is fully built. Just make it look good with CSS.

---

## What's on the page

### Top section — Join
- A QR code that players scan to open the game on their phone
- The game URL printed below it as text (backup if QR doesn't scan)
- Both update automatically when the server sends the real IP address

### Player list
- Shows everyone who has joined so far, updates in real time
- Displays the count in the heading

### Start Game button
- Disabled until at least 2 players have joined
- Label updates to show how many players are ready
- Once clicked, it disappears and the game begins

### Game state panel
Only one section is visible at a time — switches automatically as the game progresses.

| State | When it shows |
|---|---|
| Waiting | Before the host presses Start |
| Question | During each round — shows the question and all four answer options |
| Round End | After each round — shows the correct answer and current scores |
| Game Over | After all questions — shows the winner and final scores |

## Element IDs (don't rename these)

The JS finds elements by ID. Renaming any of these will break the page.

| ID | What it is |
|---|---|
| `qrcode` | Container where the QR code image is injected |
| `game-url` | The URL text shown below the QR code |
| `player-count` | The number in the "Players (N)" heading |
| `player-list` | The `<ul>` that gets filled with player names |
| `start-btn` | The Start Game button |
| `start-msg` | Text shown below the button after the game starts |
| `state-waiting` | The waiting state div |
| `state-question` | The question state div |
| `q-number` | Question number shown during a round |
| `q-text` | Question text shown during a round |
| `ans-A` `ans-B` `ans-C` `ans-D` | The four answer `<li>` elements |
| `state-round-end` | The round end state div |
| `correct-answer` | The correct answer letter shown after a round |
| `score-rows` | The `<tbody>` for the mid-game scoreboard |
| `state-game-over` | The game over state div |
| `winner-name` | The winner's name on the final screen |
| `final-rows` | The `<tbody>` for the final scoreboard |

You can add classes, wrap elements in new divs, change the surrounding structure — just keep the IDs the same.

---

## Notes for styling

- This page runs on a **laptop screen**, not a phone, so you have more room than index.html
- The QR code is 200×200px by default — you can resize the container around it but the QR image itself is generated by a library
- The scoreboard tables have a `border="1"` attribute on them — you'll probably want to remove that in CSS and style them properly
- The Start Game button being disabled vs enabled is handled in JS — you can style the `:disabled` state in CSS to make it visually obvious
